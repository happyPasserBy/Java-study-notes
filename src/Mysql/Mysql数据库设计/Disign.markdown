# Mysql数据库的设计
## 1.常见的设计规范
### 1.1 命名规范
* 所有数据库表列等名称尽量全部用小写，单词用 _ 连接
* 数据库的列名不要使用Mysql关键字 
* 数据库的列名做最好做到见名知意，并且最好不要超过32个字符
* 临时表必须以tmp为前缀并以日期为后缀，备份表以bak为前缀日期为后缀
* 所有存储相同数据的列名和列类型必须一致，如用户表的用户ID与订单表的用户ID
### 1.2 基本设计规范
* 建议使用Innodb存储引擎
* 数据库和表的字符集统一使用UTF8
* 所有的表和注释添加注释
* 尽量控制单表数据量的大小，建议控制在500万以内
* 谨慎使用Mysql分区表
* 尽量做到冷热数据分离，减小表的宽度
* 禁止在表中建立预留字段
* 禁止在数据库中存储图片文件等二进制数据
* 禁止在线上做数据库的压力测试
* 禁止从开发环境，测试环境直接连接生产环境
### 1.3 索引设计规范
* 限制每张表上的索引数量，建议单重表的索引不找过5个，索引数量也是和列的数量成正比
* 每个Innodb表必须有一个主键，如果没有Innodb会查找第一个非空唯一索引，如果也没有则会自动创建隐式主键
* 建议不要使用更新频繁的列作为主键，不用试多列主键
* 索引列的顺序
    * 区分度高的列放在联合索引的最左侧
    * 尽量把字段长度小的列放在联合索引的左侧
    * 使用频繁的列放到联合索引的左侧
* 避免建立冗余和重复索引
* 对于频繁的查询有限考虑使用覆盖索引
* 尽量避免使用外键约束，但建议在表与表之间的关联键上建立索引
### 1.4数据库字段设计规范
* 优先选择符合存储需要的最小数据类型
* 对于非负数采用无符号进行存储
* 避免使用TEXT、BLOB数据类型或者建议将TEXT、BLOB进拆表存储
* 避免使用ENUM数据类型
* 尽可能把所有列定义为NOT NULL
* 使用TIMESTAMP或DATETIME类型存储时间
* 与金额相关的必须使用decimal
### 1.5 SQL开发规范
* 建议使用预编译语句进行数据库操作
* 避免数据类型的隐式转换
    * 隐式转换会导致索引失效
* 充分利用表上已经纯在的索引
    * 避免使用双%号的查询条件，如 like '%111%'
    * 一个SQL只能利用到符合索引中的一列进行范围查询
    * 使用left join或者not exists来优惠not in,not in 会导致索引失效
* 连接不同的数据库应使用不同的账号，禁止跨库查询
* 禁止使用select *
* 禁止使用不含字段列表的INSERT语句，如insert into t values(?,?,?,?);
* 避免使用子查询，可以把子查询优化为join操作
* 避免使用JOIN关联过多表
    * 会产生临时表，影响查询效率
* 禁止使用order by rand()进行随机排序
    * 会把所有符合条件的数据加载到内存中
* WHERE从句禁止对列进行函数转换和计算
    * 对列进行转换和计算会导致无法使用索引
* 在明显不会有重复值时使用UNION ALL而不是UNION
    * UNION会把所有数据加载到临时表在进行去重操作
### 1.6 数据库操作行为规范
* 超过百万的批量写操作，要分批多次进行操作
    * 大批量操作可能会导致严重的主从延迟
    * binlog日志为row格式会产生大量的日志
* 对大表使用pt-online-schema-chane修改表结构
* 禁止为程序使用的账号赋予super权限
* 对于程序连接数据库账号遵循权限最小原则
    * 不允许一个账号跨库操作
    * 程序使用的账号不允许有drop权限

## 2.5 DB规划
### 2.5.1 DB建议
* 为以后数据库迁移提供方便
* 避免跨库操作，把经常一起关联查询的表放到一个DB中
* 为了方便识别表所在对的DB，把表名前增加数据库名称
### 2.5.2 用户数据库(mc_customerdb)
* customer_inf、customer_login、customer_level_inf、customer_login_log、customer_balance_log、customer_point_log
### 2.5.3 商品数据库(mc_productdb)
* product_info、product_pic_info、product_category、product_supplier_info、product_comment、product_brand_info
### 订单数据库(mc_orderdb)
* order_master、order_detail、order_cart、shipping_info、warehouse_proudct、warehouse_info、order_customer_addr
